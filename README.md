# CNC加工 ERP（一期）

给CNC对外加工厂用的订单与生产进度管理系统。局域网网页应用：装在公司一台电脑上，
其他电脑和连公司WiFi的手机用浏览器访问。

## 一期功能

- **订单录入**：客户采购单录入，明细行含编号、图号、品名、规格、材质、数量、单价、备注
- **PDF导入**：新建订单时可上传客户PDF采购单（含国产ERP导出的特殊编码PDF），自动识别明细表格填入表单（人工核对后保存）；系统按客户记住表格格式；图片型PDF自动转OCR识别
- **Excel台账批量导入**：订单管理页上传《客户交期明细表》，自动识别各月份工作表、按「客户+客户单号」分组成订单，新客户自动建档（不重复），客户单号已存在的自动跳过防重复导入
- **PDF+台账 双证录入**：同时上传客户PDF采购单+Excel台账，系统按PO号在台账里定位对应订单，交叉核对 PO/客户/**每个图号的件数**（逐行比对，金额不参与——台账内部不填金额也能录）；哪个图号差几件红字精确指出，PDF里有台账没有的行也拦；全部对上才准录入，以台账为准并把PDF自动留档为附件
- **作废订单自动清理**：作废的订单保留15天后自动彻底删除（含附件）；总经理/CNC主管可在作废后直接删除
- **已结案订单授权删除**：只有总经理能删除已出货结案的订单，且删除时必须重新输入登录密码授权确认；进行中的订单不能直接删除
- **价格保密**：单价和金额只有 总经理 / 财务 / CNC主管 三种角色能看到，其他角色（文员、跟单、编程、外发）从服务器就拿不到价格数据
- **单件追踪**：每一块板有独一无二的板件号（格式：`年月-订单流水-件序`，如 `2607-0015-03`），逐件报工，数量绝不糊涂
- **板件查询**：独立页面，搜图号/规格/材质/板件号/客户，直接列出每一块具体的板材（在哪个订单、现在什么状态、卡了几天），可按工序阶段筛选，点一块跳到所在订单
- **工序流转**：铣磨 → CNC → 精磨 → 电镀 → 回厂 → 出货
- **开工标记**：勾选板件点「开工」，标记"铣磨中 / CNC加工中（可备注机台）/ 精磨中"；报工完成或外发后自动清除；订单列表显示"加工中N件"
- **外发管理**：三种外发——CNC加工外发、精磨外发（磨床厂）、电镀外发，记录厂家、发出/预计回厂日期、备注；在外清单随时看谁家压着货。回货登记后状态自动流转（CNC回货→标CNC完成、精磨回货→标精磨完成、电镀回货→标电镀回厂），不用手动改状态
- **外发单打印（采购订单版）**：按本厂原版外发单复刻——公司抬头、供应商/我方信息、明细表（名称/图号/规格/材质/数量，备注列带板件号）、红字加工要求、8条条款、供应商回签栏；A4横版打印；单价/金额留空手写（对客户的单价绝不会出现在给厂家的单据上）
- **加工要求分模板预填**：加工外发与电镀外发各有一套红字要求模板（系统设置里改），开单时自动预填到弹窗、可按本单改（如这单镀8μm那单10μm），打印在外发单红字区
- **外发交接单**：装车用交接单（板件清单+装车勾选框+实际装车数签字栏），司机按单点件，对账以我方单据为准；打印页与外发单一键互切
- **外发两段式确认**：新开的外发单是"待确认"状态（板件被占住但不算在外），货实际发出后在打印页点「确认已外发」才变在外；没发成可一键撤销，板件恢复原状态
- **外发撤件**：开单5件实际只拉走4件？把没拉走的从单里撤出，恢复待外发状态，下次拉货开新单，账实永远一致
- **出货与结案**：按批出货、打印送货单；全部出货自动结案，少一件都关不了单
- **跨月追踪**：订单不出够货就一直在"进行中"列表里，不管拖多久
- **图纸附件（按图号对账）**：订单详情里每个图号单独一行，传没传图纸一眼看清（红「未传图纸」/绿「已传N个」），行内直接传；顶部汇总「图纸 X/Y」；不属于图号的文件（客户PO留档等）归入「订单附件」，可一键归到某个图号
- **看板**：进行中订单、各工序完成件数、在外提醒、交期临近提醒
- **停滞预警**：超过7/14/30天没有任何记录、又不在外发中的板件，自动在看板上标红——专治跨月订单被遗忘
- **单件履历**：订单里点任何一个板件号，看它从下单到出货的完整时间线，每一步是谁在什么时候录的都有据可查
- **应收账款（独立财务台账）**：与订单数据完全无关的手工账，财务自己记（可含模具钢材等CNC之外的业务）——客户、业务类型、事由、应收/已收金额、约定收款日；🔔一键标记催款（黄色高亮排最前，收清自动取消）；顶部汇总应收/已收/未收/催款中金额。**权限锁死（服务端逐接口强制）：财务=查看+全部操作；总经理=只读；其余角色（含采购主管）一律不可见不可操作**
- **车辆提醒**：车辆档案（车牌/年检到期日/商业保险到期日），到期前30天黄色提醒、**前14天红色报警**（不允许过期，过期更是红色叹号），看板首页同步显示到期清单
- **每日自动备份**：数据库每天自动备份到 `data/backups/`，保留最近30份
- **断电保护**：数据库开启 WAL + synchronous=FULL，每笔操作提交时强制写入磁盘——界面上显示"保存成功"的数据断电也不丢，最多丢"还没点保存的"；上传的图纸/PDF同样落盘即刷盘；服务器关机/重启时自动做数据库检查点后安全关库
- **前端零计算**：所有筛选、状态判定、滞留分级、金额汇总都在服务器算好，浏览器只负责显示——车间的旧电脑、手机开着也不吃CPU内存

## 怎么启动

双击 **启动ERP.bat**。窗口开着系统就在跑，关掉窗口系统就停。

浏览器打开 `http://localhost:3000`。局域网其他电脑用 `http://这台电脑的IP:3000`
（查IP：命令行输入 `ipconfig`，看"IPv4 地址"）。

**初始账号：`admin` / `admin123`（总经理角色，登录后请立即改密码！）**

然后在「用户管理」里给每个人建账号、分角色。

## 电脑重启后自动启动（可选）

按 `Win+R` 输入 `shell:startup` 回车，把 `启动ERP.bat` 的快捷方式拖进去即可。

## 局域网访问不了？

多半是Windows防火墙挡了3000端口。用管理员身份打开PowerShell执行一次：

```
netsh advfirewall firewall add rule name="CNC ERP" dir=in action=allow protocol=TCP localport=3000
```

## 统一门户（与刀具库存系统同一个80端口）

ERP 与刀具系统（CNCInventory 仓库）共用一个入口：浏览器打开 `http://CT的IP/`（80端口）
出现门户首页，选「订单生产 ERP」（`/erp/`）或「刀具库房管理」（`/tools/`）。
80端口的网关由刀具系统的 frontend 容器承担，通过共享 docker 网络反代到 ERP 容器。

部署/升级步骤（CT 上，两个项目目录各自执行）：

```bash
docker network create factory-net   # 只需执行一次，已存在会报错，忽略即可
cd CNC-ERP && docker compose up -d --build
cd CNCInventory && docker compose up -d --build
```

说明：
- ERP 自身的 3000 端口仍可直连（`http://CT的IP:3000` 自动跳到 `/erp/`），网关挂了也不影响 ERP 单独使用
- 老书签（如 `:3000/orders`）自动 302 跳转到新地址，扫码/收藏不用改
- 两个系统的数据库完全独立、互不影响（ERP=SQLite，刀具=MySQL），本次融合不动任何数据

## 部署到服务器（Docker，推荐）

比放办公室电脑上稳：服务器24小时开机，Proxmox 快照/备份一并保护。

1. 把整个 ERP 文件夹拷到服务器上跑 Docker 的那个 CT 里（`node_modules` 和 `web/dist` 不用拷，构建时自动生成）
2. **迁移数据**：把 Windows 这边的 `data` 文件夹整个拷到服务器项目目录下（里面是数据库+图纸附件+备份，拷完 Windows 这边就别再录了，以服务器为准）
3. 在项目目录执行：
   ```
   docker compose up -d --build
   ```
4. 浏览器访问 `http://CT的IP:3000`。以后升级：替换代码文件后再执行一次上面这条命令，`data` 目录挂在容器外面，数据不受影响。

### 国内网络：默认已配好镜像源

**docker-compose 默认就走国内源**（基础镜像走 DaoCloud、npm 走 npmmirror），国内主机上直接
`docker compose up -d --build` 即可，什么都不用配。想换回官方源：
`NODE_IMAGE=node:24-alpine NPM_REGISTRY=https://registry.npmjs.org docker compose build`。

代码本身没有"墙外私有下载"的坑：依赖全部来自 npm 仓库（无编译、无 GitHub Release 下载），
OCR 词库（`server/tessdata/`）和 PDF 字体映射都打包在仓库/依赖里，**运行时完全不需要联网**。

- **git clone 慢**：重试几次一般能过；不行就在本机 clone 好整个文件夹拷过去（不含 node_modules 和 web/dist）
- **不用 Docker 裸跑时**：`npm config set registry https://registry.npmmirror.com` 再 `npm ci`

### 零下载迁移：本机构建好的镜像直接拷到服务器

服务器网络再差也不怕——在已经构建成功的电脑上导出镜像，服务器一个字节都不用下载：

```bash
# 本机（Windows PowerShell 也一样）：导出镜像（约200MB压缩包）
docker save cnc-erp:latest -o cnc-erp.tar

# 把 cnc-erp.tar、docker-compose.yml、data文件夹 拷到服务器同一目录，然后：
docker load -i cnc-erp.tar
docker compose up -d --no-build
```

以后升级也走这条路：本机构建 → save → 拷过去 → load → `docker compose up -d --no-build`。

时区已在镜像里设为 Asia/Shanghai（影响单号年月和备份日期，别去掉）。
数据库就是 `data/erp.sqlite` 单文件（SQLite），这个规模不需要单独的数据库服务；
除了系统自带的每日备份（`data/backups/`），建议 Proxmox 层面对这个 CT 做定期备份，双保险。

## 数据在哪里

- 数据库：`data/erp.sqlite`（整个系统的数据就这一个文件，复制它=完整备份）
- 图纸附件：`data/uploads/`
- 自动备份：`data/backups/`

换电脑时把整个 ERP 文件夹拷过去（需要装 Node.js 22 以上），双击启动即可。

### 统一登录（与刀具系统）

在 ERP 登录过的浏览器，进刀具系统（`/tools/`）自动免密登录，角色对应：
总经理→刀具总经理、采购主管→刀具采购主管、CNC主管→刀具CNC主管、财务→刀具总经理（只读报表）。
文员/跟单/编程/外发默认不开通刀具系统（需要的话让管理员在刀具系统单独建号）。
刀具系统里已有的同名账号沿用其本地角色，不会被覆盖。反过来（先登刀具再进ERP）不免密，ERP 要单独登录。

## 技术说明（给以后维护的人）

- 后端：Node.js + Express，数据库用 Node 内置 SQLite（无需安装数据库）
- 前端：Vue 3 + Element Plus，源码在 `web/`，改了前端要重新打包：`cd web && npm run build`
- 服务端口改 `PORT` 环境变量，默认 3000
- 二期候选：询价报价、材料采购（余量计算）、月度对账单、扫码报工
