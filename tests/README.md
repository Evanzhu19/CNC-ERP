# 测试套件

五套自动化测试，随时可以自己跑。**默认打 3000 端口。security 不造数据、正式库上跑也安全；
其余四套会造测试数据，请在开发实例上跑**（`node server/dev.js` 起在别的端口，再传端口号）。

```bash
npm test                # 一键跑全套业务+安全（regress → business → chaos → security）

npm run test:regress    # 业务全流程回归 34 项
npm run test:business   # 业务铁律 45 项（数量守恒/防重复发货/工序顺序…）
npm run test:chaos      # 破坏性 23 项（并发抢占/重复提交/事务回滚…）
npm run test:security   # 安全 8 类（越权/泄露/注入/畸形输入）—— 正式库可跑
npm run test:perf       # 性能压测（造100单3000板，只在开发实例跑）

node tests/business.mjs 5173    # 指定端口
```

## 各自守什么

| 套件 | 覆盖 | 为什么重要 |
|------|------|-----------|
| **business** 45项 | **数量守恒**（订单10件→板件10件→出货必须刚好10件、少一件关不了单）、**一块板不能出两次货**、**一块板不能同时在两张外发单**（真实事故：开单5件实际拉走4件）、**工序不能跳**（未CNC完不能电镀）、**出货必须电镀回厂**、**撤销必须逆序**、**外发中禁止改状态**、回货自动流转、特殊状态拦截出货、电镀厂直送、作废/结案保护 | 每一条都是真金白银的规矩，破一条就可能少发货、多发货、货款算错 |
| **chaos** 23项 | 网络卡了猛点10次（只记1条）、出货猛点5次（只成功1次、不重复发货）、两人同抢同一批板开外发单（只1张成功）、乱序点旧页面按钮、并发删除+操作、22个操作洪水、**事务回滚**（一批混了不合格件，合格的也不许偷偷出货） | 车间真实的"人祸"：网卡、手抖、两个人抢着操作 |
| **regress** 34项 | 录单→报工→外发→回货→出货→结案 全流程 + 图纸附件 + 同图号序号 + 刃长提示 | 主干流程不能坏 |
| **security** 8类 | 价格越权（4角色×3接口）、写权限越权、财务隔离（4角色）、提权、无效token、SQL注入、18种畸形输入不致500、错误信息不泄露数据库结构 | 角色之间不能越权，不能泄露 |
| **perf** | 100单3000板下的看板/列表/查询/并发读写 | 数据涨上来不能卡 |
