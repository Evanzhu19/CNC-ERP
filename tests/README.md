# 测试套件

三套自动化测试，随时可以自己跑。**默认打 3000 端口（正式系统）——正式库上只跑 security，
regress 和 perf 会造测试数据，务必在开发实例上跑。**

```bash
npm run test:security   # 安全：越权/信息泄露/注入/畸形输入/崩溃防护
npm run test:regress    # 业务回归：录单→报工→外发→回货→出货全流程（会造数据！）
npm run test:perf       # 性能：造100张订单3000块板，压热点接口（会造数据！）

# 打开发实例（先 node server/dev.js 起在别的端口）
node tests/security.mjs 5173
```

## 各自查什么

| 套件 | 覆盖 |
|------|------|
| `security.mjs` | 8类：价格越权（4个不可见价格的角色×3个接口）、写权限越权、财务数据泄露（4角色）、提权/改他人账号、无效token、SQL注入、18种畸形输入不致500、错误信息不泄露数据库结构 |
| `regress.mjs` | 34项：全流程 + 工序顺序强制 + 外发两段式 + 撤件/撤回货 + 特殊状态禁出货 + 同图号序号 + 刃长提示 + 按图号传图纸 |
| `perf.mjs` | 100订单/3000板下的看板、订单列表、板件查询、并发读、并发写锁冲突 |

## 改完代码后

至少跑 `npm run test:security` 和 `npm run test:regress`（开发实例上），
两个都全绿再部署到 CT。security 全绿的标准是「发现问题 0 个」。
