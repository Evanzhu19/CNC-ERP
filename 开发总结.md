# CNC-ERP 开发总结

> 东莞市瑞宏智能科技有限公司 内部生产管理系统
> 开发时间：2026-07-05 ~ 2026-07-07（持续开发中）

## 一、这个系统解决什么问题

瑞宏智能是 CNC 对外加工厂，主接机械大板和自动化机架加工。核心痛点：

1. **跨月追踪出错**：订单工期长（常跨数月）、新单不断，人工（Excel台账）追踪时老订单被遗忘、数量记岔
2. **外发环节丢账**：半夜外发电镀，开单 5 件实际拉走 4 件，下次对方再开一张单，对账吃亏
3. **价格保密**：单价/金额只能让总经理、财务、CNC主管看到，车间和文员不能接触

## 二、业务流程（系统按此建模）

```
客户询价(带图纸) → 报价 → 客户PO → 材料采购(加余量) → 铣磨部门(铣边铣面)
→ CNC编程/加工(2.5米×2、4米、6米、8米龙门；产能不够外发CNC厂)
→ 磨床精磨(4米/6米；也会外发磨床厂) → 外发电镀(镀铬) → 回厂 → 打包出货
```

一期范围：订单录入 → 生产流转 → 外发 → 出货核对（询价报价、材料采购、月度对账留二期）。

## 三、核心设计决策

| 决策 | 理由 |
|---|---|
| **单件追踪**：每块板有唯一板件号 `年月-订单流水-件序`（如 2607-0015-03） | 用户要求"一个不能少"；同图号多件必须逐件区分 |
| **完成+进行中双状态**：报工记录"做完了什么"，开工标记记录"正在做什么" | 车间要能表达"正在铣/本厂CNC加工中(6米机)/精磨中" |
| **外发两段式**：开单=待确认(draft，板被占住但不算在外) → 人工点"确认已外发"才算在外 | 浏览器测不到打印成功；防止开了单没发货就记账 |
| **价格字段级权限**：服务端直接不下发单价/金额给无权角色 | 前端隐藏可被绕过，服务端过滤才可靠 |
| **SQLite 单文件库**：不上独立数据库服务 | 十来人规模绰绰有余；备份=复制文件；Node 内置 node:sqlite 零依赖 |
| **订单不出够货永不沉底** + 停滞预警（N天无动静标红） | 直击跨月遗忘痛点 |

## 四、功能清单

### 订单
- 订单/明细/单件三级结构，板件号自动生成；返回按钮、列宽自适应
- **PDF 导入**：客户 PDF 采购单自动识别明细填表。支持国产 ERP 导出的 CID/GBK 编码 PDF（需 pdfjs cMap），纯图片 PDF 自动转 OCR（tesseract.js，词库离线打包）。表头关键词词典 + 按客户记忆列映射（pdf_mappings 表）
- **Excel 台账批量导入**：识别《客户交期明细表》多月份 sheet（表头逐月变化、客户列合并单元格、混合日期格式），按"客户+客户单号"分组建单；客户自动建档不重复、单号查重防重复导入
- 作废订单保留 15 天自动彻底删除，可手动直删（总经理/CNC主管）
- **已结案订单删除需总经理授权**（重输登录密码）

### 生产
- 报工（铣磨/CNC/精磨完成，可撤销）；开工标记（铣磨中/CNC加工中+机台备注/精磨中，报工或外发自动清除）
- 单件履历时间线：每一步谁在何时录入，点板件号即查

### 外发（三种：CNC加工 / 精磨 / 电镀）
- 两段式：draft 待确认 → 确认在外 → 回货登记（回货自动标对应工序完成）
- 交接单打印（装车勾选框 + 实际装车数手写栏 + 签字），对账以我方单据为准
- 撤件（开单5件拉走4件的纠错）、撤单；在外清单显示在外天数/超预计回期
- 厂家支持多类型（一家厂可同时做 CNC+磨）

### 出货与看板
- 按件出货、送货单打印（价格可选打印）、全部出货自动结案、少一件关不了单
- 看板：进行中订单、工序漏斗、加工外发/电镀在外（含超期）、**停滞预警**（超7/14/30天无动静且不在外的板标红）、交期临近
- 打印单据统一使用系统设置里的厂名+LOGO（客户与厂家→系统设置）

### 权限（7角色）
总经理/财务/CNC主管可见价格；文员/跟单可录入；编程/外发只读；用户管理仅总经理。

## 五、技术栈与结构

- **后端**：Node.js 24 + Express 5 + node:sqlite（内置，零外部数据库）
- **前端**：Vue 3 + Element Plus + Vite，构建后由后端静态托管
- **文件解析**：pdfjs-dist（文字提取+渲染）、tesseract.js（OCR）、@napi-rs/canvas（PDF转图）、xlsx（Excel）
- **部署**：Docker（多阶段构建，TZ=Asia/Shanghai），`docker compose up -d --build` 一条命令；数据挂载在容器外 `./data`

```
server/                 后端
  index.js              入口：静态托管、每日备份、作废清理定时任务
  db.js                 建表 + 迁移（rebuildTable 处理 CHECK 约束变更）+ 种子
  auth.js               会话、角色、价格权限
  routes-basics.js      登录/客户/厂家/用户/系统设置(厂名LOGO)
  routes-orders.js      订单/明细/单件/PDF与Excel导入/附件/删除授权
  routes-production.js  报工/开工/外发(draft-open-done)/出货/看板
  pdf-parse.js          PDF表格重建 + 表头词典（PDF/Excel共用）
  pdf-ocr.js            OCR兜底（页面渲染+tesseract）
  excel-parse.js        台账解析（多sheet/前向填充/分组）
  tessdata/             离线OCR词库(chi_sim+eng)
web/src/views/          页面：看板/订单列表/录入/详情/外发/打印×2/基础资料/用户
data/                   (不进仓库) erp.sqlite + uploads/ + backups/ + logo
```

## 六点五、第二批改进（2026-07-07）

**流程硬约束**
- 工序强制顺序 铣磨→CNC→精磨→电镀→出货，报工/开工/外发/出货全部校验前置工序（修复"CNC未完可电镀外发"bug）
- 撤销强制逆序：只能撤最后一道完成的工序；外发中的板禁止撤工序（修复bug）；新增回货撤销补全逆序链
- 出货必须电镀回厂后（唯一例外：电镀厂直送）

**外发增强**
- 外发类型改为逗号分隔多工序：支持铣磨外发、组合外发单（CNC+精磨同厂连做，须相邻工序），回货按包含工序依次自动标完成
- 厂家多能力（含铣磨），外发厂家列表按"须具备全部所选工序能力"过滤
- 工序格显示本厂完成日期 + 「外」上标（外发回来的），一眼分本厂/外发
- **电镀厂直送客户**：电镀外发单可对未回货的板一步完成"电镀回厂+出货"，送货单标注直送，按订单自动分单/结案

**信息与预警**
- 订单列表搜索扩展：规格、材质、品名、板件号（原图号搜索确认可用）
- 状态色彩：灰待/蓝厂内加工/橙在外/紫特殊状态/绿完成/黄红滞留
- 明细行备注列（修复导入备注不可见）+ 单件备注（可随时改，进履历）
- 特殊状态 维修/返工/改图：紫色标记+说明，禁止出货，进履历，看板统计；退工序走人工逆序撤销
- 滞留分级预警：状态无变化 ≥2天黄/≥4天红（阈值系统设置可调），在外的板同样计算，看板/订单列表/详情三处高亮

**关键实现点**
- `pieces` 加 note/flag/flag_note/flag_date/flag_by；`outsourcing.type` 去掉CHECK约束改多值CSV
- 顺序/撤销/回货逻辑集中在 routes-production.js 的 STAGE_ORDER/PREREQ/parseProcs/hasStage/laterDoneStage
- "当前状态开始时间"由 max(各工序recorded_at, 外发created_at, 回货returned_date, wip_date, flag_date) 计算
